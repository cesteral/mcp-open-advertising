## 11. Audience

**Area URL:** `/api/area/Audience`

### GQL `audience` Query

Get an existing audience by ID.

```graphql
audience(id: ID!): Audience!
```

**Return Type: `Audience`**

| Field               | Type                                  | Nullability  | Description                                          |
| ------------------- | ------------------------------------- | ------------ | ---------------------------------------------------- |
| activeUniques       | ActiveUniques                         | NULLABLE     | Active unique user counts.                           |
| advertiser          | Advertiser!                           | NON-NULLABLE | The advertiser that owns the audience.               |
| createdAt           | DateTime!                             | NON-NULLABLE | Date/time in UTC the audience was created.           |
| description         | String                                | NULLABLE     | Description of the audience.                         |
| id                  | ID!                                   | NON-NULLABLE | Unique ID.                                           |
| isAutoGenerated     | Boolean!                              | NON-NULLABLE | Whether automatically generated by the platform.     |
| lastUpdatedAt       | DateTime!                             | NON-NULLABLE | Last updated time (UTC).                             |
| name                | String!                               | NON-NULLABLE | The name of the audience.                            |
| targetEveryone      | Boolean!                              | NON-NULLABLE | If true, this is a TargetEveryone audience.          |
| targetingDataGroups | AudienceTargetingDataGroupsConnection | NULLABLE     | Targeting data groups associated with this audience. |

### GQL `audiences` Query

Get existing audiences.

```graphql
audiences(after: String, before: String, first: Int, last: Int, order: [AudienceSortInput!], where: AudienceFilterInput): AudiencesConnection
```

**Return Type: `AudiencesConnection`**

| Field      | Type             | Description              |
| ---------- | ---------------- | ------------------------ |
| edges      | [AudiencesEdge!] | List of edges.           |
| nodes      | [Audience!]      | Flattened list of nodes. |
| pageInfo   | PageInfo!        | Pagination info.         |
| totalCount | Int!             | Total count of items.    |

### REST `POST /v3/audience`

Create a new Audience.

**Key Request Fields:**

| Field                | Type       | Required | Description                                                           |
| -------------------- | ---------- | -------- | --------------------------------------------------------------------- |
| AdvertiserId         | string     | REQUIRED | Platform ID of the owning advertiser.                                 |
| AudienceName         | string(64) | REQUIRED | Name of the Audience.                                                 |
| IncludedDataGroupIds | string[]   | REQUIRED | Data Groups that include a user (intersection — user must be in ALL). |
| Description          | string     | NULLABLE | Optional description.                                                 |
| ExcludedDataGroupIds | string[]   | optional | Data Groups that exclude a user from the audience.                    |
| TargetEveryone       | boolean    | optional | If true, creates a TargetEveryone audience.                           |

### REST `PUT /v3/audience`

Update an existing Audience.

### REST `GET /v3/audience/{audienceId}`

Get an existing Audience by ID.

### REST `GET /v3/audience/{audienceId}/{geoForCounts}`

Get an existing Audience by ID, and filter the returned counts by geo.

### REST `GET /v3/audience/availableaudienceacceleratorexclusions`

Retrieve the third-party data ID and name of segments that can be added to the audience accelerator exclusions list of an ad group.

### REST `POST /v3/audience/insights/appssites`

Retrieve app and site audience insights for the specified audience and geographic areas. Returns a score representing how relevant an interest is for the audience.

### REST `POST /v3/audience/insights/browser`

Retrieve browser insights. Returns a breakdown of each browser as a percentage of the audience.

### REST `POST /v3/audience/insights/channelmix`

Retrieve channel mix insights for the specified audience and geographic areas.

### REST `POST /v3/audience/insights/device`

Retrieve device insights for the specified audience and geographic areas.

### REST `POST /v3/audience/insights/geo`

Retrieve geo location audience insights. Returns a score representing how relevant a location is for the audience.

### REST `POST /v3/audience/insights/interests`

Retrieve interest audience insights. Returns a score representing how relevant an interest is for the audience.

### REST `POST /v3/audience/insights/marketmix`

Retrieve market mix. Returns a breakdown of each market type as a percentage of available inventory matching the audience.

### REST `POST /v3/audience/insights/os`

Retrieve browser operating system insights. Returns a breakdown of each OS as a percentage of the audience.

### REST `POST /v3/audience/insights/privatecontract`

Retrieve private contract audience insights.

### REST `POST /v3/audience/insights/thirdparty`

Retrieve third-party data audience insights.

### REST `GET /v3/audience/name/{audienceId}`

Get the identifier and name for an Audience based on identifier.

### REST `POST /v3/audience/query/advertiser`

Query for a page of Audiences within an Advertiser.

### REST `GET /v3/audience/query/facets`

Get the facets of Audiences that can be queried.

## 12. Authentication

**Area URL:** `/api/area/Authentication`

### REST `POST /v3/authentication`

Create and retrieve a short-lived API token with an expiration of up to 24 hours (1440 minutes).

> To create a long-lived API token, use the interactive UI through the Manage API Tokens option under the User Profile menu in the Developer Portal.

**Request Body:**

| Field                    | Type    | Required | Description                                        |
| ------------------------ | ------- | -------- | -------------------------------------------------- |
| Login                    | string  | REQUIRED | The username (email address) for the account.      |
| Password                 | string  | REQUIRED | The password for the account.                      |
| TokenExpirationInMinutes | integer | optional | Token expiration in minutes (max 1440 = 24 hours). |

**Response (HTTP 200):**

| Field          | Type              | Description                                                  |
| -------------- | ----------------- | ------------------------------------------------------------ |
| Token          | string            | The authentication token to use in the Authorization header. |
| ExpirationDate | string<date-time> | Expiration date/time of the token.                           |


### GQL `bidListCreate` Mutation

Create a bid list.

```graphql
bidListCreate(input: BidListCreateInput!): PayloadWithErrorsOfBidList!
```

### GQL `bidListUpdate` Mutation

Updates a bid list. Lets you set lines to add and lines to remove — does not require specifying all bid lines.

### GQL `bidListSet` Mutation

Updates a bid list by setting ALL lines to what is in the input (replaces entire list).

### GQL `bidListDelete` Mutation

Deletes a bid list.

### GQL `adGroupAssociateBidList` Mutation

Associate or remove bid lists from an ad group and enable or disable them for bidding.

### GQL `advertiserAssociateBidList` Mutation

Associate or remove bid lists from an advertiser and enable or disable them for bidding.

### REST `POST /v3/bidlist` ⚠️ DEPRECATED

> Use `bidListCreate` mutation in GraphQL instead.

Create a bid list, returning the created bid list.

### REST `PUT /v3/bidlist` ⚠️ DEPRECATED

> Use `bidListUpdate` mutation in GraphQL instead.

Update the bid list for the given ID.

### REST `POST /v3/bidlist/batch` ⚠️ DEPRECATED

> Use `bidListCreate` mutation in GraphQL instead.

Create multiple bid lists at once.

### REST `PUT /v3/bidlist/batch` ⚠️ DEPRECATED

> Use `bidListUpdate` mutation in GraphQL instead.

Update multiple bid lists at once.

### REST `POST /v3/bidlist/batch/get`

Retrieve multiple bid lists by IDs.

### REST `GET /v3/bidlist/{bidListId}` ⚠️ DEPRECATED

> Use the `bidList` query in GraphQL instead.

Retrieve the content of a bid list including all bid dimension line items.

### REST `DELETE /v3/bidlist/{bidListId}` ⚠️ DEPRECATED

> Use `bidListDelete` mutation in GraphQL instead.

Delete a user bid list and its associations.

### REST `POST /v3/bidlist/query/unassociated`

Retrieve a paged list of unassociated bid lists at a given level of ownership.

### REST `POST /v3/bidlistsummary/query/adgroup/available` ⚠️ DEPRECATED

Retrieve a paged list of bid lists available to associate with the provided ad group.

### REST `POST /v3/bidlistsummary/query/advertiser/available` ⚠️ DEPRECATED

Retrieve a paged list of bid lists available to associate with the provided advertiser.

### REST `POST /v3/bidlistsummary/query/campaign/available` ⚠️ DEPRECATED

Retrieve a paged list of bid lists available to associate with the provided campaign.

### REST `POST /v3/bidlistsummary/query/global`

Retrieve a paged list of global bid lists. By default, bid lists including a private contract ID are excluded.

### REST `POST /v3/bidlistsummary/query/partner/available` ⚠️ DEPRECATED

Retrieve a paged list of bid lists available to associate with the provided partner.

## 15. Campaign Flight

**Area URL:** `/api/area/Campaign%20Flight`

### GQL `campaignFlightCreate` Mutation

Create a campaign flight using the data specified in the input.

### GQL `campaignFlightUpdate` Mutation

Updates a campaign flight using the data specified in the input.

### GQL `campaignFlightDelete` Mutation

Deletes a campaign flight using the campaign flight ID provided in the input.

### REST `DELETE /v3/campaignflight/{campaignFlightId}` 🔶 LEGACY

> Use `campaignFlightDelete` mutation in GraphQL instead.

Delete an existing campaign flight.

**Restrictions:**

- Only permits deleting past flights if it is the last flight for the campaign.
- Cannot delete the only flight in a campaign.
- Cannot delete if the flight has already started spending.
- Cannot delete a past flight (end date in the past).

## 31. Delta

**Area URL:** `/api/area/Delta`

Delta endpoints allow you to track what has changed since the last known version, enabling efficient synchronization.

### GQL `adGroupDelta` Query

Get all changed ad groups for an advertiser or campaign since the last change-tracking version.

### GQL `campaignDelta` Query

Get all changed campaigns for an advertiser since the last change-tracking version.

### GQL `creativeDelta` Query

Get all changed creatives for an advertiser since the last change-tracking version.

### GQL `trackingTagDelta` Query

Get all changed tracking tags for an advertiser since the last change-tracking version.

### REST `POST /v3/delta/activity/query/adgroup`

Retrieve a list of IDs of all activity logs for the specified ad group that were updated since the last change tracking version.

> TIP: Use GraphQL for improved performance. See also the Delta Python scripts and Platform Synchronization guide.

### REST `POST /v3/delta/adgroup/query/advertiser` 🔶 LEGACY

> Use `adGroupDelta` query in GraphQL instead.

Retrieve a list of IDs of ad groups modified since the last change tracking version.

### REST `POST /v3/delta/advertiser/query/partner`

Retrieve a list of IDs of advertisers modified since the last change tracking version.

### REST `POST /v3/delta/bidlist/query/global`

Retrieve a list of IDs of global bid lists modified since the last change tracking version.

### REST `POST /v3/delta/bidlist/query/partner`

Retrieve a list of IDs of partner bid lists modified since the last change tracking version.

### REST `POST /v3/delta/campaign/query/advertiser` 🔶 LEGACY

> Use `campaignDelta` query in GraphQL instead.

Retrieve a list of IDs of campaigns modified since the last change tracking version.

### REST `POST /v3/delta/contract/query/advertiser/available`

Retrieve a list of IDs of contracts available to an advertiser that were modified since the last change tracking version.

### REST `POST /v3/delta/creative/query/advertiser` 🔶 LEGACY

> Use `creativeDelta` query in GraphQL instead.

Retrieve a list of IDs of creatives modified since the last change tracking version.

### REST `POST /v3/delta/dmp/query/advertiser/firstparty`

Retrieve a list of IDs of all first-party data segments modified since the last change tracking version.
